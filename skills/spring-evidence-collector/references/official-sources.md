# Official Sources

Checked on: 2026-07-12. These official vendor documents are normative for the behavior they describe; re-check version-sensitive pages before use.

- [Maven POM reference](https://maven.apache.org/pom.html) — inheritance, dependency management, effective POM, and dependency-tree limitations.
- [Maven security model](https://maven.apache.org/security) — build files, plugins, dependencies, and repositories as trusted execution inputs.
- [Maven multiple modules](https://maven.apache.org/guides/mini/guide-multiple-modules.html) — reactor module collection and ordering.
- [Gradle dependency reports](https://docs.gradle.org/current/userguide/viewing_debugging_dependencies.html) — resolved dependency and dependency-insight reports.
- [Gradle version catalogs](https://docs.gradle.org/current/userguide/version_catalogs.html) — catalog aliases and version declarations.
- [Gradle securing builds](https://docs.gradle.org/current/userguide/security.html) — build logic and plugin supply-chain boundaries.
- [Gradle Wrapper](https://docs.gradle.org/current/userguide/gradle_wrapper.html) — distribution checksums, Wrapper JAR integrity, download, and credential risks.
- [Gradle build lifecycle](https://docs.gradle.org/current/userguide/build_lifecycle.html) — init, settings, plugin, configuration, and task execution phases.
- [Gradle initialization scripts](https://docs.gradle.org/current/userguide/init_scripts.html) — global and user-home scripts that can affect every build before project configuration.
- [Spring Boot externalized configuration](https://docs.spring.io/spring-boot/reference/features/external-config.html) — property sources and precedence.
- [Spring Boot Actuator endpoints](https://docs.spring.io/spring-boot/reference/actuator/endpoints.html) — endpoint exposure and sensitive-value sanitization.
- [Spring Boot configuration metadata](https://docs.spring.io/spring-boot/specification/configuration-metadata/index.html) — configuration key metadata format.

Static file parsing is intentionally narrower than these build and runtime models.
